# Benchmark Reports

Store reproducible benchmark plans and concise result summaries here.

Each report should include:

- Git commit and branch.
- Machine, OS, JVM, Docker, and container resource limits.
- Dataset size, warm-up, measured duration, concurrency, and request mix.
- Application and dependency configuration.
- Exact commands used to run the benchmark.
- Throughput, P50/P95/P99, error rate, dependency impact, and known variance.

Do not commit raw large logs, local databases, secrets, or screenshots as the only
evidence.
